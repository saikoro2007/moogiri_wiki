# みんなで大喜利公式Wiki (Moogiri Wiki)

Discordサーバー「みんなで大喜利」の公式ルールやガイドラインを管理するリポジトリです。
このサイトは [Docusaurus](https://docusaurus.io/) を使用して構築されており、GitHub Pagesを通じて自動公開される仕組みになっています。

## 管理者向け：ページを編集・追加する方法

このWikiは**非エンジニアの方でもブラウザから直接編集可能**です。
以下の手順に従って、Wikiの内容を更新してください。

### 1. 編集したいファイルを探す
1. このGitHubリポジトリの `docs/` フォルダを開きます。
   - ルールに関する内容: `docs/rules/`
   - FAQに関する内容: `docs/faq/`
2. 編集したい `.md`（Markdown）ファイルをクリックします。

### 2. ファイルを編集する
1. ファイルの右上にある **鉛筆マーク（Edit this file）** をクリックします。
2. テキストエディタが開くので、内容をプレーンテキストで編集します。
   - `## 見出し` や `- 箇条書き` など、簡単なMarkdown記法が使えます。
   - 上部の **Preview** タブをクリックすると、プレビュー画面で崩れていないか確認できます。

### 3. 保存して更新依頼を出す (Pull Request)
1. 編集が終わったら、画面右上の **"Commit changes..."** という緑色のボタンを押します。
2. 出てきたウィンドウの「Commit message」に「〇〇のルールを追加」「脱字の修正」など、**何をしたか**を簡単に書きます。
3. "Create a new branch for this commit and start a pull request" を選び、そのまま **"Propose changes"** をクリックします。
4. 次の画面でそのまま **"Create pull request"** を押せば完了です！

> 💡 変更を直接公開するのではなく「Pull Request（レビュー依頼）」という形にすることで、万が一壊れてしまっても管理者が確認した上で反映できるようになっています。

## 開発者向け情報

### ローカルでの動作確認

手元でプレビューを確認したい場合は、Node.js (v18以上推奨) が必要です。

```bash
# インストール
npm install

# 開発サーバーの起動（ホットリロード対応）
npm start
```

ビルドテスト（エラーチェック）は以下で行います。
```bash
npm run build
```
